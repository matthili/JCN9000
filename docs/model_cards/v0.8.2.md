## Was diese Iteration verbessert

Identische zwei Hebel wie bei Kreuz (Solo nutzt dieselbe Determinisierung +
denselben Full-Round-Lookahead):

1. **Void-aware Determinisierung** ([`jass_engine/void_inference.py`](../../jass_engine/void_inference.py))
   -- behebt das "Trumpf-Ziehen gegen blanke Gegner" (systematischer Bias).
2. **60 statt 30 Rollouts** -- weniger Label-Rauschen.

Die Solo-Heuristik (Schwierigkeitsgrad + Eval-Gegner) wurde ebenfalls getunt
(staerker als zu v0.8.1).

## Architektur

Unveraendert (identisch zu Kreuz/v0.8.1): MLP 768/768/384, `MaskBias`, 421-dim
State + 36-dim Mask, Policy + Value. Nicht austauschbar mit dem Kreuz-Modell
(andere Reward-Struktur: eigene Punkte statt Team-Punkte).

## Bewertung

Eval-Setup: paired-eval mit zyklischer Sitz-Rotation, `batched-gpu`,
Spielziel-Mix 500/1000. Im Solo sitzen pro Tisch 1 Modell A + 1 Modell B +
2 Solo-Heuristiken (H) -- die "interessante" Baseline ist 25 % (vier gleich
starke Spieler).

| Vergleich | v0.8.2 (A) | Gegner (B) | Fueller (2x H) | Partien |
|---|---|---|---|---|
| vs. solo_mcts2 | **46,8 %** | 40,3 % | je ~6,5 % | 4000 |
| vs. Heuristik | **78,8 %** | 7,5 % | je ~6,9 % | 4000 |

Beim Modell-zu-Modell-Vergleich gewann v0.8.2 von den 3481 NN-entschiedenen
Partien **53,7 %** -- bei dieser Stichprobe ~4,4 Standardabweichungen ueber 50 %,
also ein bestaetigter Gewinn (der erste 1000er-Lauf hatte 46,2 % vs. 41,0 %
gezeigt). Gegen drei Heuristiken 78,8 % (gegen v0.8.1's 77,2 %, jetzt aber vs.
staerkerer Heuristik).

**Punktverlauf vs. Heuristik:** Avg. Score 520,0 vs. 310,3; Matsch-Rate
0,50 % vs. 0,09 %.

## Bekannte Schwaechen

- Greedy-Inference; Trumpf-Ansage/Weisen via Heuristik; keine Mensch-Validierung.
- "vs. Heuristik"-Zahlen nicht direkt mit v0.8.1 vergleichbar (Heuristik getunt).
  Sauberer Vergleich ist das Modell-zu-Modell-Gate.

## Reproduktion

WSL2 mit GPU, conda-Env `jass-gpu`:

```bash
python -u -m training.data.generate_solo_mcts_data_mp \
    --warm-start models/solo_mcts2/best.keras \
    --games-per-variant 500 --games-per-chunk 25 \
    --rollouts-per-card 60 --target-distribution "500:0.5,1000:0.5" \
    --workers 8 --parallel-threads-per-worker 32 \
    --inference-batch-size 1024 \
    --skip-existing \
    --output data/solo_mcts_fixed/phase3

python -u -m training.train \
    --data data/solo_mcts_fixed/phase3 \
    --warm-start models/solo_mcts2/best.keras \
    --output models/solo_mcts3 \
    --epochs 20 --learning-rate 5e-4

python -m evaluation.run_solo_eval --a nn --b nn \
    --model-a models/solo_mcts3/best.keras \
    --model-b models/solo_mcts2/best.keras \
    --games 4000 --paired-eval --inference-mode batched-gpu \
    --inference-batch-size 128 --parallel-threads 128
```
