## Was diese Iteration verbessert

**Full-Round-Lookahead.** Bisher nutzte die Bodensee-MCTS-Datengen nur einen
1-Stich-Lookahead -- der Lehrer plante buchstaeblich nur einen von 18 Stichen
voraus und spielte die hintere Rundenhaelfte quasi blind. Neu spielt
[`bodensee_vectorized_lookahead.py`](../../training/data/bodensee_vectorized_lookahead.py)
pro Kandidaten-Karte die **gesamte Restrunde** aus (vektorisiert, mit
Forced-Move-Abkuerzung) und bewertet sie an der Punkte-Differenz bis
Rundenende -- inklusive Letzter-Stich-Bonus und Matsch. Damit sieht der Lehrer
erstmals Reihenfolge-Entscheidungen wie "erst die Schrott-Karte werfen, dann
selbst stechen, um den +5-Bonus des letzten Stichs mitzunehmen".

Begleitend wurde die Bodensee-Heuristik (Ansage-Fallback -- der **auch in den
NN-Partien der App** ansagt) per Parametersuche getunt.

## Trainingsverfahren

| Parameter | MCTS-Runde 3 |
|---|---|
| Warm-Start | bodensee_mcts2 |
| Lernrate | 5e-4 |
| Epochen | 40 (Early-Stopping bei 35) |
| Beste Val-Policy-Acc | 52,63 % |
| Beste Val-Value-MAE | **0,2035** (Runde 2: 0,2540) |

**Wichtig zur Val-Policy-Accuracy (52,6 % vs. 84,4 % bei mcts2):** Das ist
**kein** Qualitaetsverlust. Die beiden Zahlen messen *verschiedene Lehrer*. Der
1-Stich-Lehrer war fast deterministisch ("welche Karte gewinnt diesen Stich"
hat meist eine eindeutige Antwort) -> leicht zu imitieren -> 84 %. Der
Full-Round-Lehrer steht staendig vor **Beinahe-Gleichstaenden** (im Endspiel
fuehren oft mehrere Karten zum identischen Rundenergebnis) und "wuerfelt"
zwischen gleichwertigen Karten -- diesen Muenzwurf kann kein Modell vorhersagen.
Dass die Daten dennoch signalreich sind, zeigt der **bessere Value-Head**
(0,254 -> 0,204) und vor allem das vernichtende Eval-Ergebnis. Der Richter ist
das Spiel, nicht die Imitations-Genauigkeit.

Die 40 Epochen (statt frueher Abbruch wie bei Kreuz/Solo) waren noetig, weil
der Full-Round-Lehrer ein groesserer Sprung weg von der single-trick-trainierten
Warm-Start-Politik ist.

## Bewertung

Eval-Setup: paired-eval, `batched-gpu`, Spielziel-Mix 500/1000. 2-Spieler-
Baseline ist 50 %.

| Vergleich | v0.9.2 Win-Rate | Gegner | Partien |
|---|---|---|---|
| vs. bodensee_mcts2 (single-trick-Vorlaeufer) | **92,4 %** | 7,6 % | 8000 |
| vs. Heuristik (getunt) | **96,8 %** | 3,2 % | 4000 |

Der Sprung ist beispiellos: ueber alle 10 vorkommenden Varianten **91-99 %**,
extrem gleichmaessig -- die Signatur einer *systematischen* Ueberlegenheit, nicht
eines Zufalls. Bei 8000 Partien liegt der Modell-zu-Modell-Vorsprung ~140
Standardabweichungen ueber 50 %. Avg. Score vs. Heuristik 550,0 vs. 344,4.

Zur Einordnung: bodensee_mcts2 schlug seinen Vorlaeufer nur mit 53,0 % -- der
1-Stich-Lehrer war ausgereizt. Full-Round hat das Plateau gesprengt.

## Bekannte Schwaechen

- **Die Heuristik ist als Massstab gesaettigt.** Mit 96,8 % verliert sie
  praktisch immer -- sie taugt fuer Bodensee nicht mehr als Messlatte. Der
  naechste aussagekraeftige Test ist **menschliches Spiel**.
- **`i_am_announcer`-Bit im Encoder** ist im Datengen weiterhin auf `False`
  fixiert (faktisch tot) -- unveraendert ggue. v0.9.x.
- Greedy-Inference; Ansage via Heuristik; keine Mensch-Validierung der Zahlen.
- Slalom kommt im Bodensee nicht vor (Heuristik sagt es nicht an).

## Reproduktion

WSL2 mit GPU, conda-Env `jass-gpu`:

```bash
# MCTS-Runde 3 -- erstmals --lookahead-mode full-round
python -u -m training.data.generate_bodensee_mcts_data_mp \
    --warm-start models/bodensee_mcts2/best.keras \
    --games-per-variant 500 --games-per-chunk 25 \
    --rollouts-per-card 60 --target-distribution "500:0.5,1000:0.5" \
    --workers 8 --parallel-threads-per-worker 32 \
    --inference-batch-size 1024 \
    --lookahead-mode full-round \
    --skip-existing \
    --output data/bodensee_mcts_fixed/phase3

python -u -m training.train \
    --data data/bodensee_mcts_fixed/phase3 \
    --warm-start models/bodensee_mcts2/best.keras \
    --output models/bodensee_mcts3 \
    --epochs 40 --learning-rate 5e-4

python -m evaluation.run_bodensee_eval --a nn --b nn \
    --model-a models/bodensee_mcts3/best.keras \
    --model-b models/bodensee_mcts2/best.keras \
    --games 8000 --paired-eval --inference-mode batched-gpu \
    --inference-batch-size 128 --parallel-threads 128
```
