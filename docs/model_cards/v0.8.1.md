## Begruendung des Re-Trainings

Identisch zu v0.7.1: `legal_moves` musste "bedienen ODER stechen" zulassen,
hat aber nur "bedienen + Buur" zugelassen. Da MCTS-Datengen und Heuristik-
Player auf derselben Engine fussen, sind alle v0.8.0-Trainingsdaten deprecated.

## Architektur

Identisch zu v0.8.0 und v0.7.x: MLP 768/768/384, `MaskBias`, 421-dim State +
36-dim Mask, Policy + Value. Nur die Gewichte sind Solo-spezifisch.

**Wichtig:** Nicht austauschbar mit Kreuz-Jass-Modellen. Die Reward-Struktur
ist fundamental anders (eigene Punkte statt Team-Punkte). Web-App muss
anhand der Spielart das richtige Modell laden.

## Bewertung

Eval-Setup: paired-eval, 1000 Partien, Spielziel-Mix 500/1000 (passend zur
Trainings-Verteilung), `batched-gpu`. In jedem Spiel: 1 Modell + 3
Heuristik-Bots (Solo ist 4-Spieler; Random-Baseline = 25 %, sehr starke
Heuristik = ~12-15 %).

| Vergleich | v0.8.1 Win-Rate | Gegner Win-Rate |
|---|---|---|
| vs. solo_mcts1 (eigener Vorlaeufer) | **47,2 %** vs. 35,4 % (Rest: 2× Heuristik 8,7 %) | -- |
| vs. Heuristik (alle drei Mitspieler) | **77,2 %** | 7,1 % bzw. 7,8 % |

**Interpretation:** Im 4-Spieler-Spiel ist die "interessante" Baseline 25 %
(vier gleichstarke Spieler). solo_mcts2 mit 47,2 % gegen ein nicht-triviales
MCTS-Vorlaeufer-Modell und 77,2 % gegen reine Heuristiken ist eine sehr
deutliche Verbesserung. Die 77,2 % entsprechen ca. 3-mal so vielen Siegen
wie ein zufaelliger Tisch-Genosse erwartet bekaeme.

**Punktverlauf:**

| Metrik | v0.8.1 vs. mcts1 | v0.8.1 vs. Heuristik |
|---|---|---|
| Avg. Score pro Partie | 463,2 | 513,6 |
| Avg. Runden pro Partie | 8,4 | 7,8 |
| Matsch-Rate pro Runde | 0,16 % | 0,43 % |

**Pro Variante** (Win-Rate v0.8.1 vs. drei Heuristiken):

| Variante | Win-Rate | Stichprobe (Runden) |
|---|---|---|
| Trumpf Eichel | 84,4 % | 64 |
| Trumpf Schelle | 74,1 % | 116 |
| Trumpf Herz | 70,8 % | 48 |
| Trumpf Laub | 82,1 % | 56 |
| Gumpf Eichel | 79,2 % | 120 |
| Gumpf Schelle | 70,5 % | 88 |
| Gumpf Herz | 76,9 % | 104 |
| Gumpf Laub | 78,0 % | 100 |
| Oben | 68,4 % | 76 |
| Unten | 81,1 % | 228 |

Stark gleichmaessig zwischen ~68 % und ~84 %. Schwaechste: Oben (n=76, kleine
Stichprobe -- die 68,4 % sind innerhalb des Rauschens vergleichbar mit Gumpf).
Staerkste: Trumpf Eichel (n=64, ebenfalls klein).

## Bekannte Schwaechen

- **Trumpf-Ansage und Weisen** werden vom Solo-Heuristik-Bot uebernommen,
  nicht vom NN.
- **Greedy-Inference:** Modell ist vorhersehbar, weil argmax.
- **Kleine Sub-Varianten-Stichproben.** Trumpf Herz (n=48), Trumpf Eichel
  (n=64) sind statistisch wackliger als die anderen.
- **Mensch-vs-Modell-Validierung steht aus** -- die Zahlen sind gegen das
  eigene Vorlaeufer-Modell und gegen die Heuristik gemessen.

## Reproduktion

WSL2 mit GPU, conda-Env `jass-gpu`:

```bash
# Solo-MCTS-Runde 1 -- Warm-Start: v0.5.0-Modell (Kreuz, vor dem Engine-Bug)
python -u -m training.data.generate_solo_mcts_data_mp \
    --warm-start models/v5/best.keras \
    --games-per-variant 150 --games-per-chunk 25 \
    --rollouts-per-card 30 --target-distribution "500:0.5,1000:0.5" \
    --workers 8 --parallel-threads-per-worker 32 \
    --inference-batch-size 1024 \
    --output data/solo_mcts_fixed/phase1

python -u -m training.train \
    --data data/solo_mcts_fixed/phase1 \
    --warm-start models/v5/best.keras \
    --output models/solo_mcts1 \
    --epochs 20 --learning-rate 5e-4

# Solo-MCTS-Runde 2
python -u -m training.data.generate_solo_mcts_data_mp \
    --warm-start models/solo_mcts1/best.keras \
    --games-per-variant 500 --games-per-chunk 25 \
    --rollouts-per-card 30 --target-distribution "500:0.5,1000:0.5" \
    --workers 8 --parallel-threads-per-worker 32 \
    --inference-batch-size 1024 \
    --output data/solo_mcts_fixed/phase2

python -u -m training.train \
    --data data/solo_mcts_fixed/phase2 \
    --warm-start models/solo_mcts1/best.keras \
    --output models/solo_mcts2 \
    --epochs 20 --learning-rate 5e-4

# Eval
python -m evaluation.run_solo_eval \
    --a nn --b heuristic \
    --model-a models/solo_mcts2/best.keras \
    --games 1000 --paired-eval \
    --inference-mode batched-gpu
```
